## Debugging Workflow

1. **Reproduce** - Confirm the bug with a minimal, reliable reproduction
2. **Isolate** - Narrow the failure to the smallest possible scope (single function/module)
3. **Hypothesize** - Form a specific hypothesis before changing code
4. **Verify** - Test the hypothesis with a targeted log or assertion, not trial-and-error changes
5. **Fix** - Make the minimal change that resolves the root cause
6. **Harden** - Add a regression test that would have caught this bug
